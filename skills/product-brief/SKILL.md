# Product Brief Workflow (Clara Muse)

## Context
Translating Marcus's vision into a technical blueprint for Viktor.

## Logic Steps
1. **Synthesize Concept**: Read Marcus's Product Concept.
2. **Define User Stories**: Write "As a user, I want X so that Y".
3. **Establish Functional Specs**: List exactly what the code must do.
4. **Define Non-Functional Specs**: List constraints like performance (e.g., "Must load in under 2 seconds").
5. **Acceptance Criteria**: Define exactly what tests must pass for Viktor to hand it to Elias.

## Output
Product Requirements Document (PRD).
